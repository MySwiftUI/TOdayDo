## 작업 방식 📝
1. 이슈 생성
2. feature 브랜치 생성 (ex. feature/3(이슈번호))
3. PR 생성
4. 해당 PR 리뷰
5. Merge 진행
6. 코드 리뷰 반영 이슈 생성
7. 코드 리뷰 PR
8. 코드 리뷰 반영
> 코드 리뷰가 정해진 시간에 진행되는 것이 아니라 다른 UI 및 기능 개발을 위해 PR 따로 진행
</br>
